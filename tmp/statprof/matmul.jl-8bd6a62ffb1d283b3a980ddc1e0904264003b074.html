1128 """
1129 *(A, B::AbstractMatrix, C)
1130 A * B * C * D
1131
1132 Chained multiplication of 3 or 4 matrices is done in the most efficient sequence,
1133 based on the sizes of the arrays. That is, the number of scalar multiplications needed
1134 for `(A * B) * C` (with 3 dense matrices) is compared to that for `A * (B * C)`
1135 to choose which of these to execute.
1136
1137 If the last factor is a vector, or the first a transposed vector, then it is efficient
1138 to deal with these first. In particular `x' * B * y` means `(x' * B) * y`
1139 for an ordinary column-major `B::Matrix`. Unlike `dot(x, B, y)`, this
1140 allocates an intermediate array.
1141
1142 If the first or last factor is a number, this will be fused with the matrix
1143 multiplication, using 5-arg [`mul!`](@ref).
1144
1145 See also [`muladd`](@ref), [`dot`](@ref).
1146
1147 !!! compat "Julia 1.7"
1148 These optimisations require at least Julia 1.7.
1149 """
